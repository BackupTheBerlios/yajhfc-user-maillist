This is a known Java bug, too:
http://bugs.sun.com/bugdatabase/view_bug.do?bug_id=6544857

According to the bug entry above a possible workaround is as follows:
| You can workaround/fix this problem by modifying the registry.
|
| The relevant key is:
|HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\User 
Shell Folders
|
| A couple of the Shell Folders ("Personal" and "My Pictures" in my case) had 
an absolute path to a drive that no longer existed.
| The folders should be %USERPROFILE%\Pictures rather than absolute paths.

It would be nice if you could try if this helps in your case (and write a 
short message if it does/does not).
If it doesn't help I can try implementing a workaround in YajHFC, but I that 
would take some time to do (so I would like to avoid that if possible).
