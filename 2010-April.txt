I checked this again and it seems that this column really is always empty for 
sent faxes (because HylaFAX does not supply a value for it). For faxes in 
the "Transmitting" tab it shows the scheduled time, however.

In general the problem is that there seems to be no documentation from HylaFAX 
about the different columns/fields, so I have resort to "trial and error" to 
find out which columns are reasonable to display and which are not.

So, in future version I will probably change the default layout of the Sent 
tab to not include this column any more.

The difference between sendfax and YajHFC is that YajHFC (depending on the 
settings) creates one PDF file from the input using GhostScript on the 
client, while sendfax uploads the multiple files unconverted.

You can get the same behaviour as sendfax from YajHFC if you go to the 
Options->"Paths and Viewers" and select "Multiple files; no conversion" from 
the "Send multiple files as" combo box. With that option, YajHFC will never 
invoke gs and upload the PS/PDF files to HylaFAX server as they are.

Apart from that, I think this is certainly a GhostScript problem since YajHFC 
does not modify the PostScript or PDF itself, but only passes it through 
GhostScript to create one PDF file.

I was unable to reproduce the problem here so far, however, so I cannot say 
more at the moment...


Hope that helps

This problem indicates a permission problem on the HylaFAX server.
To be able to delete a fax, the user logged in to the server must have "write" 
rights on the recvq directory or be in admin mode.

HylaFAX uses the same *UNIX* user for all users, the only difference can be in 
the UNIX group. 
Which group a user uses is set by the "uid" field in the file hosts.hfaxd (-u 
parameter of faxadduser). The HylaFAX uid that is set there corresponds to 
UNIX gids (group IDs).

For you that means, you will have the following choices:
1. Set the group of the recvq directory to the gid that corresponds to your 
HylaFAX users
2. Set the permissions of the recvq directory to 0777 if any user should be 
allowed to delete faxes.
3. Go into admin mode if you want to delete faxes.

If any reader of this list knows of another possibility, please let me know.

Hope that helps
Jonas
