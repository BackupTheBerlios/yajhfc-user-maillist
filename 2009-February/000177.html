--- On Fri, 2/20/09, member horvath <member at horvath.co.za> wrote:

> Hi,
> 
> There is an option within yajhfc to choose a modem, but for
> large
> deployments this can be an administritive burden, i'm
> not sure if you
> can restrict a user to ONLY use that modem however.
> To assign users to a secific modem in hylafax itself is
> very easy, but
> again to explicitly restrict them from using another
> I'm not sure,
> although I'm sure it can be done.
> 
> To assign a modem to a user do the following:
> (Adjust paths to suite your installation)
> 
> On the hylafax server edit:
> /var/spool/hylafax/etc/config
> ADD
> JobControlCmd:          etc/jobcontrol
> 
> Then create the file:
> /var/spool/hylafax/etc/jobcontrol
> ADD
> #!/bin/sh
> . etc/setup.cache
> . bin/common-functions
> QFILE=sendq/q$1
> parseQfile
> case "$owner" in
> root) echo "Modem: any";;
> user1) echo "Modem: ttyS0";;
> user2) echo "Modem: ttyS1";;
> *) echo "Modem: ttyS2";;
> esac
> exit 0
> Make sure you then configure the clients to log into the
> hylafax
> server with the relevant user from above script (Perhaps
> you split
> your users into relevant groups and name them accordingly)
> Also make sure your /var/spool/hylafax/etc/hosts.hfaxd has
> the allowed users in.
> 
> Hope this helps

It definitely helps a lot! Especially the sample script.
