Am Donnerstag, 26. Februar 2009 schrieb p.rinaldi at comune.livorno.it:
> I have installed hylafax server and on my desktop YajHFC client 4.0.
> In received tab i see the owner of job is a number (14) not a hylafax user
> (prinaldi). I set UID of faxadduser of hylafax at 355 the rule is: when
> sent to xxx xxx355 mail to prinaldi at .... where i set 355=prinaldi?

Generally the mapping between UIDs and user names is done in hosts.hfaxd in 
HylaFAX.

It seems, however, that the HylaFAX server does not map UIDs back to user 
names when it lists the received faxes, so you see a numeric ID in YajHFC.

Unfortunately, it seems that there is no way for a HylaFAX client to resolve  
a UID into a user name (or a user name into a UID), so there is not much I 
can do about that in YajHFC (BTW: If someone knows how to do such a 
resolution, please let know).
